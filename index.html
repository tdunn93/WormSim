WormSim

WormSim - A computer worm propagation simulator; enables combinations of different known worms, attack & defense methods and network topologies to be configured in order to simulate how worms may behave in different environments

The basic three levels represent fictitious university mesh networks of varying size, the custom level allows for arbitrary configuration of different topologies and attack types to be used.

Represents an uninfected node.

Represents an incapacitated node.

Represents a node infected with W32.ILOVEYOU. ILOVEYOU attacked tens of millions of computers by spreading an email message with the subject line "ILOVEYOU" and the attachment "LOVE-LETTER-FOR-YOU.txt.vbs". In this case the visual basic extension was hidden, the worm used social engineering to lead users to think it was an ordinary text file, upon opening the script it did damage to local files and propagated itself further by email harvesting.

Represents a node infected with W32.BLASTER. Blaster exploited a vulnerability on windows server 2003, it then opened a UDP port and began to upload its viral payload before looking for new machines to infect.

Represents a node infected with W32.BORM. Borm was among one of the first worms to attack a backdoor compromised system, it made use of a remote administration facility called back orifice to infect targets.

Represents a node infected with W32.SLAMMER. Slammer is a type of worm who's payload is used to carry out DDoS attacks.

A Distributed denial of service attack (DDoS), is an attack carried out by two or more nodes aimed at incapacitating another node.

A smurf DDoS is a type of distributed denial of service attack. The smurf is installed on a node via a worms payload, it then attempts to ping all nodes on the network using a the source address of the infected node, this causes all the computers to respond to the node at once overwhelming it with traffic.

Represents a node infected with W32.STORM. Storm is a type of worm used to carry out distributed denial of service attacks by creating a (storm) botnet

A storm botnet is a remotely controlled network of computers that are infected with W32.STORM, it can be used to carry out large scale DDoS attacks by targeting nodes reachable from those infected with W32.STORM.

Honey pots are strategic defence mechanisms which lure in worms by redirecting their target locators from compromised nodes, minimising network damage.

Intrusion detection systems monitor network traffic for suspicious activity, if a suspicious packet is found it is dropped.
